WHY THIS TABLE EXISTS (standards-basis gap, not "just a characterisation figure"): the paper's PHY
anchor 802.11bd IS an OFDM system whose cross-subcarrier coding carries FREQUENCY DIVERSITY. The digital
branch was previously evaluated only on AWGN + FLAT Rayleigh; the "feature-level infeasible under
Rayleigh" sub-finding rested on no-diversity flat fading, which a comms reviewer can dismiss as a
strawman against 802.11bd. OFDM-LDPC is the scientifically interesting MIDDLE case between flat-Rayleigh
(diversity order 1, all-dead in 0-20 dB) and AWGN (sharp cliff): frequency diversity should re-open a
usable high-SNR window.

MODEL (fixed once, ONE profile -- no profile sweep, that is another paper):
- OFDM: 802.11bd 10 MHz numerology -> N_FFT=64, SCS Delta_f=156.25 kHz, N_SC=52 data subcarriers.
- Channel: frequency-selective TDL, exponential power-delay profile, L=8 taps at uniform delays
  0..3*tau_rms, per-CODEWORD-INDEPENDENT complex tap gains g_l ~ CN(0, P_l), Sum P_l = 1 so the average
  subcarrier power E[|H[k]|^2]=1 (matches the flat-Rayleigh Es/N0 normalisation). SAME coherence
  assumption as the flat-Rayleigh table: independent per codeword (coherence ~1-3 ms << frame OTA time),
  no HARQ, no cross-frame interleaving. Coded symbols frequency-INTERLEAVED (symbol i -> subcarrier
  i mod N_SC), the standard 802.11 interleaver, so one 1000-bit codeword spans the N_SC subcarrier gains
  = frequency diversity of order ~L. Perfect per-subcarrier CSI (per-symbol effective N0 = N0/|H[k]|^2).
- tau_rms = 46.2 ns : READ FROM 3GPP TR 37.885 (ATIS.3GPP.37.885.V1530), the below-6-GHz eV2X channel
  model delay-spread table. Urban NLOSv row: mu_lgDS = -0.4*log10(1+fc) - 7, sigma_lgDS = 0.1, with
  lgDS = log10(DS/1s). At fc = 5.9 GHz (802.11bd): mu_lgDS = -0.4*log10(6.9) - 7 = -7.3355 =>
  DS(median) = 10^(-7.3355) = 46.2 ns (1-sigma range 36.7-58.2 ns). Scenario = Urban NLOSv (LOS blocked
  by vehicles, same street), matching OPV2V urban intersections + the occlusion frames where feature-
  level help is most valuable. [Corrected from a placeholder 100 ns; table re-run at 46.2 ns.]
- PHYSICS SELF-CHECK (frequency selectivity must span the codeword, else the diversity mechanism is
  vacuous): coherence bandwidth B_c ~ 1/(5*tau_rms) = 1/(5*46.2 ns) ~ 4.3 MHz. 802.11bd SCS = 156.25 kHz;
  the codeword is frequency-interleaved across N_SC=52 subcarriers = 8.125 MHz > B_c. So the codeword
  spans ~8.125/4.3 ~ 1.9 coherence bands (diversity order ~2): frequency selectivity DOES span the
  codeword. PASS -- the OFDM diversity mechanism is real, not vacuous.

RESULT (tau_rms=46.2 ns, TR 37.885 Urban NLOSv @ 5.9 GHz): 16-QAM frame-BLER first feasible at ~24 dB
(24.0 dB frame 0.998 -> 28 dB 0.448). Three-channel cliff comparison (16-QAM, frame feasible from):
  AWGN 8.0 dB  |  OFDM 24 dB  |  flat Rayleigh NEVER (0-24+ dB, frame BLER=1).
=> DIVERSITY ORDER DETERMINES FEASIBILITY: no-fading (AWGN) 8 dB -> freq-selective OFDM (order ~2) 24 dB
-> no-diversity flat Rayleigh infinity. The pre-registered sub-finding upgrade holds. NOTE the OFDM window
(24 dB) sits ABOVE the selector's operational SNR range U[0,20], so within-range OFDM-LDPC is infeasible
like Rayleigh -- but a CLIFF EXISTS (Rayleigh has none). This belongs to Figure A / the BLER cliff
comparison (feasibility evidence), NOT the U[0,20] two-regime edge (selector-value evidence). Merged into

DIVERSITY ORDER PROVENANCE (supervisor requirement -- no unmeasured order): empirical diversity order
from a log-log slope fit of the 16-QAM codeword BLER over its waterfall region (BLER_cw in [1e-4, 0.1]):
d = -10 * slope of log10(BLER_cw) vs Es/N0(dB). Results: OFDM d = 2.01 (23 pts, 14.5-28 dB) -> "empirical
diversity order ~2 (BLER slope fit)"; flat Rayleigh d = 0.84 (~1, 10 pts, 16.5-21 dB) = theoretical order
1; AWGN has no waterfall in that band (no fading, effectively unbounded order). The coherence-bandwidth
self-check (~1.9) and the slope fit (2.01) agree. Feasibility threshold monotone in diversity order:
AWGN 8 dB (no fading) / OFDM ~24 dB (order 2) / flat Rayleigh unbounded (order 1).

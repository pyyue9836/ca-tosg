RECIPE
  GT source   : the OpenCOOD evaluation-time GT = post_processor.generate_gt_bbx
                (base_postprocessor.py L45-96): every CAV's object_bbx_center projected to the ego
                frame, UNIONED and DEDUPED by object_id, filtered to GT_RANGE. The canonical GT is the
                intermediate (comp) all-CAV union GT -- the fuller cooperative task-truth union
                (late GT is the same union path but a slightly-truncated batch under its ±70.4 config).
  GT_RANGE    : [-140, -40, -3, 140, 40, 1]  (standard OPV2V benchmark evaluation range; x±140, y±40).
  predictions : NOT cropped. Q4 (gt_audit.csv) confirmed late predictions include |x|>70.4 boxes
                (collaborator detections projected to ego), so there is NO structural range asymmetry
                and the ±70.4-crop idea is rejected. Far-range late-vs-comp = collaborator boxes vs
                collaborator features = the L-vs-C comparison itself, not unfairness.
  ruler       : ALL policies (late / comp / ego / Where2comm ref / CA-TOSG) scored against this ONE
                canonical GT; F1 at IoU 0.5, AP at IoU {0.3,0.5,0.7}. Same GT feeds the F1 and the
                AP (true_e2e) pipelines.
  visibility  : NO ≥1-lidar-point visibility filter. Because the GT is now one-for-all, visibility is
                a SENSITIVITY question, not a fairness one; no-filter matches the standard OPV2V
                protocol (zero reviewer friction). A ≥1-point union-visibility variant is an appendix
                sensitivity candidate, NOT the canonical.
  ego far     : ego-only has zero far predictions (gt_audit.csv) -> far objects are FN for ego. That is
                the physical fact (ego is blind there), retained; fallback cost is measured on task truth.
